## COMO O ASSISTENTE FUNCIONA
### Fluxo em 3 Etapas

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   1. PERGUNTA          2. BUSCA NOS DOCS          3. RESPOSTA       │
│                                                                     │
│   Atendente digita  →  IA vasculha toda a     →  Resposta clara    │
│   em linguagem         base de documentos        em linguagem      │
│   natural              internos da NovaTech      natural +         │
│                                                   FONTE CITADA     │
│   "Qual o prazo         [Manual de Processos]                      │
│    de reembolso         [Política Comercial]     "Conforme o       │
│    para cliente         [FAQ Interno]             Manual de        │
│    premium?"            ...                       Processos,       │
│                                                   seção 4.2, o     │
│                      ⚠ A IA só responde com       prazo é 7 dias." │
│                        o que está na base.                         │
│                        Base fraca = resposta                       │
│                        fraca.                                      │
└─────────────────────────────────────────────────────────────────────┘
```

> **Princípio central:** A qualidade das respostas depende diretamente da qualidade e atualização dos documentos internos da NovaTech.

---

## O QUE O ASSISTENTE FAZ MUITO BEM

| Capacidade | Descrição |
|---|---|
| **Busca instantânea** | Encontra informações em toda a base documental em segundos |
| **Linguagem natural** | Entende perguntas como um humano as faria, sem precisar de palavras-chave exatas |
| **Respostas com fonte** | Toda resposta cita o documento de origem, garantindo rastreabilidade |
| **Disponibilidade total** | Funciona 24/7, sem fila, sem variação de humor |
| **Padronização** | Todos os atendentes recebem a mesma informação, eliminando respostas inconsistentes |
| **Escalabilidade** | Atende múltiplos usuários simultaneamente sem perda de qualidade |

---

## O QUE O ASSISTENTE NÃO FAZ

| Limitação | Por quê importa saber |
|---|---|
| **Não inventa respostas** | Se a informação não está nos documentos, ele informa que não encontrou — não cria dados falsos |
| **Não acessa a internet** | É especialista nos dados da NovaTech, não um buscador geral |
| **Não toma decisões** | Fornece informação e contexto; a decisão final é sempre do atendente |
| **Não substitui documentação ruim** | Documentos desatualizados ou incompletos resultam em respostas limitadas |
| **Não garante 100% de precisão** | Atua com alta probabilidade de acerto, mas requer validação humana em casos críticos |

## CRITÉRIOS DE SUCESSO — MÊS 3

| # | Critério | Meta | Ferramenta de Medição |
|---|---|---|---|
| 1 | **Respostas com fonte verificável** | ≥ 90% das respostas citam fonte identificável | Log automático da plataforma |
| 2 | **Redução no tempo de busca manual** | ≥ 40% vs. tempo médio atual (baseline mês 1) | Pesquisa com equipe de atendimento |
| 3 | **Adoção pela equipe** | ≥ 70% dos atendentes usando ≥ 5x/semana | Analytics de uso |
